* intro
a diferença entre variáveis locais e variáveis globais/de ambiente  é que as locais tem visibilidade restrita, apenas a sessão do shell em que ela foi definida, e uma variável global tem visibilidade não só na sessão do shell em que foi definida mas também em ambientes derivados, ou seja, subshells

a forma para definir as variáveis também é diferente, onde para definir uma variável local é necessário apenas um valor a um node de variável, mas para uma variável global/de ambiente é necessário o uso do comando *export*

definindo uma variável local ->
#+begin_src bash
[13:55][]~ ✮ name=Biel
[13:55][]~ ✮ echo $name
Biel
#+end_src

definindo uma variável global/de ambiente ->
#+begin_src bash
[13:56][]~ ✮ export nome=gabriel
[13:56][]~ ✮ echo $nome
gabriel
#+end_src

para apagar uma variável ->
#+begin_src bash
unset nome

unset name
#+end_src

* set , env
o comando set e env geralmente são utilizados para visualizar variáveis, mas também podem ser usados para definir variáveis dependendo do shell atual

O comando *env* exibe apenas variáveis globais e não locais

O comando *set* demonstra as variáveis locais também

* aspas ' vs "
as aspas utilizadas para exibição de variáveis interfer diretamente no resultado a ser obtido, quando utilizamos aspas duplas *"* a variável é interpretada, e seu valor exibido, porem com aspas simples *'*, o valor da variável não é interpretado

com aspas duplas ->
#+begin_src sh
[14:37][]~ ✮ echo "ola $name"
ola gabriel
#+end_src

com aspas simples ->
#+begin_src sh
[14:37][]~ ✮ echo 'ola $name'
ola $name
#+end_src

* variáveis especiais do shell

- $! - Variável armazena o PID do último processo executado em segundo plano(background).

- $$ - Variável armazena o PID do shell atual.

- $? - Variável armazena o código de retorno do ultimo comando executado
